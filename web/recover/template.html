ShardQuorum Recovery

Offline recovery tool. Reconstructs a secret from a quorum of shards.

Self-test

Built-in vectors prove this tool recovers correctly. It runs on load.

How to use this, and how it works

You need two things:

  1. Enough shards from the same set. Look at any shard text file (shard-*.txt) — the third line tells you how many you need, e.g. "Any 3 of 5 of these shards".
  2. The recovery envelope (ur:sq-env/...). This is included with every shard — it's the last line of every shard-*.txt file and appears on every shard QR sheet.

To recover your secret:

  1. Open a shard text file (shard-*.txt) from each kit you have. Paste each shard on its own line in the Shards box above. A shard can be the printed word list (e.g. tuna next keep ...) or the ur:sskr/... string — either works.
  2. Paste the recovery envelope (the ur:sq-env/... line) into the Recovery envelope field.
  3. Press Recover secret.

Your secret appears in the output area. If recovery fails, the tool tells you why — not enough shards, a mistyped word, or shards from different secrets.

How it works: This is a single, self-contained HTML file. All computation runs on your device and it never makes a network request. You can (and should) disconnect from the internet before using it.

The algorithm behind this tool is Shamir's Secret Sharing. It is described in full in RECOVERY-SPEC.md, which ships alongside this file in every recovery kit. This tool includes built-in test vectors that prove it works correctly — run them with the Run self-test button above.
